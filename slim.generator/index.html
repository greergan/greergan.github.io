    Welcome to
    Slim.Generator

    A Typescript static website generator

    Slim.Generator is a highly configurable static website generator written in 100% pure Typescript. It uses Slim.View as its HTML rendering engine and supports both Deno and Slim Typescript platforms.

    Command line arguments

    The generate script accepts multiple command line argument.

    --config | -c path
    		Valid path to a Slim.Generator configuration file
    --generate_id | -id number
    A numeric id which specifies a Slim.Generator page filter when rendering
    Can also be controlled inside the Slim.Generator configuration file
    --help | -h
    		Displays help
    --git | -git path
    		A local valid absolute | relative | file:/// path to a Slim style git configuration file
    --verbose -v
    		Turns on verbose runtime information
    --version
    		Displays version information

    --config | -c

    The --[config | -c] flag is required. The argument that follows must be a valid file path ending in .json.

    --config path [optional_flags]

    --generate_id | -id

    The [--generate_id | -id] flag is optional and controls which pages are generated.

    This configuration option overrides the value that is found in the top level configuration if it exists. It triggers page processing when the supplied number matches a page.generate_id value.

    --config path [optional_flags] [--generate_id | -id] number
    Top level configuration property is overrided by the command line flag - 
    {
    "generate_id": number,
    ...
}
    Page level configuration property - 
    {
    "page": {
        "generate_id": number,
        ...
    }
}

    --help | -help

    The [--help | -h] flag is optional and displays the generate.ts program help information.

    [--help | -h]

    --git | -git

    The [--git | -git] flag is optional and specifies the path to a locally stored git configuration file. When specified on the command line, this option turns on git storage integration.

    This configuration cannot be part of the website configuration file if it is stored in a GitHub repository. GitHub has a process that scans repositories looking for GitHub access keys.

    --config path [optional_flags] [--git | -git] [absolute | relative | file:/// ] path
    {
    "access_keys": {
        "github": {
            "repositories": {
                "input": {
                    "user": string,
                    "repository": string,
                    "access_key": hash
                }
                "output": {
                    "user": string,
                    "repository": string,
                    "access_key": hash
                }
            }
        }
    }
}

    --verbose | -v

    The [--verbose | -v] flag is optional and enables trace information to be displayed.

    --config path [optional_flags] [--verbose | -v]

    --version

    The --version flag is optional and displays generate.ts help information.

    --version

    Configuration options

    The website_configuration.json file can contian the following options. Some options are required while others are optional.

    Top level configuration properties 
    {
    "output_to": path,
    "namespace": path,
    "generate_id": number,
    "copy_to_output": object array,
    "site": object,
    
    "pages": object array
}
    Site level configuration properties 
    "site": {
    "meta_tags": property array,
    developer_named_property: any
}
    copy_to_output level configuration properties 
    "copy_to_output": [
    {"path": string},
    {"path": string, "output": string}
],
    Page level configuration properties 
    "pages": [
    {
        "input_file": path,
        "output_file": path,
        "title": string,
        "meta_tags": property array,
        "external_models": object array
    }
]
    External_models level configuration properties 
    "external_models": [
    {"path": path },
    {"path": path }
]

    Configuration example

    The following configuration example is used to generate the Slim website.

    https://greergan.github.io/models/website.json 
    {
    "output_to": "./dist",
    "namespace": "https://greergan.github.io",
    "copy_to_output": [
        {"path": "assets"},
        {"path": "models"},
        {"path": "pages"},
        {"path": "partials"},
        {"path": "includes"},
        {"path": "../slim.generator/generator.ts", "output": "./"}
    ],
    "site": {
        "meta_tags": [],
        "copyright": {
            "by": "Jeff Greer",
            "started_date": "2023"
        }
    },
    "pages": [
        {
            "input_file": "pages/index.html",
            "output_file": "index.html",
            "title": "Slim",
            "meta_tags": []
        },
        {
            "input_file": "pages/slim.view.html",
            "output_file": "slim.view/index.html",
            "title": "Slim.View",
            "external_models": [
                {"path": "models/actors.json"},
                {"path": "models/authors.json"}
            ]
        },
        {
            "input_file": "pages/slim.generator.html",
            "output_file": "slim.generator/index.html",
            "title": "Slim.Generator"
        },
        {
            "input_file": "pages/slim.utilities.html",
            "output_file": "slim.utilities/index.html",
            "title": "Slim.Utilities"
        },
        {
            "input_file": "pages/slim.objects.html",
            "output_file": "slim.objects/index.html",
            "title": "Slim.Objects"
        },
        {
            "input_file": "pages/slim.editor.html",
            "output_file": "slim.editor/index.html",
            "title": "Slim.Editor"
        }
    ]
}
    Create an output directory and then run this working configuration from that directory.
