    Welcome to
    Slim.View

    A Typescript HTML rendering engine

    Inspired by Deno remote imports; there is now a rendering engine that supports remote view inclusion. With Slim.View it is possible to import HTML assets from either remote locations on the Web or from the local file system.

    Use Slim.View as part of a Typescript web server stack or as part of a static website stack. Slim.Generator is an example of a static site generator.

    Namespace

    A variable supplied when a new view is created. This variable is prepended to all relative paths found inside of include statements.

    The "file" protocol.

    file:///path/to/website/root

    The "http" protocol.

    http://greergan.github.io

    The "https" protocol.

    https://greergan.github.io

    Includes

    Include view templates whithin any HTML document. The "view" tag can contain either relative or absolute URLs. The view namespace is prepended to each relative path.

    Include reusable <head> template which may also contain it's own include files or other reusable site definitions. This type of template inclusion can be used in each top level page of a website.

    <include view="includes/head.html">

    Include partial view templates. This is an arbitray concent. The following examples includes small, specific, html code based on the idea that a top level file will benefit from including smaller chunks of HTML code. The following "partials" are used to build the current page you are viewing.

    <include view="partials/slim.view/variable_substitution.html">
    <include view="partials/slim.view/view_includes.html">

    Include standard document <footer>, bottom of page templates and closing of site level HTML objects.

    <include view="includes/foot.html">

    Statements

    Render views or html fragments using "with" or "foreach" statements. Statements are called using positional arguments and can span multiple lines.

    [with|foreach] model="model" view="path/to/view" | html_fragment

    with statement - model and view

    {% with model="users[1]" view="partials/slim.view/user.html" %}

    Carrie Fisher is an actor

    with statement - model and html fragment

    {%
    with model="users[2]"
        <p> {{first_name}} {{last_name}} is an {{profession}} </p>
%}

    Harry Harrison is an author

    foreach statement - model and view

    {% foreach model="users" view="partials/slim.view/user.html" %}

    Harrison Ford is an actor

    Carrie Fisher is an actor

    Harry Harrison is an author

    Robert Asprin is an author

    foreach statement - model and html fragment

    {%
    foreach model="users"
        <p> {{first_name}} {{last_name}} is an {{profession}} </p>
%}

    Harrison Ford is an actor

    Carrie Fisher is an actor

    Harry Harrison is an author

    Robert Asprin is an author
